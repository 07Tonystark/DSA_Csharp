Fixed-size contiguous memory. Fastest random access. Foundation for two pointers, sliding window, and binary search on sorted arrays.

Access O(1) Search O(n) Sort O(n log n) 
// Two pointers — find pair with target sum (sorted array)
int[] nums = { 1, 2, 4, 6, 8 };
int left = 0, right = nums.Length - 1, target = 10;
while (left < right) {
    int sum = nums[left] + nums[right];
    if (sum == target) { /* found */ break; }
    else if (sum < target) left++;
    else right--;
}

// Binary search on sorted array
int idx = Array.BinarySearch(nums, target);
if (idx < 0) idx = ~idx; // insertion point if not found
MethodComplexityDescription
arr[i]O(1)Index access
arr.LengthO(1)Fixed size
Array.Sort(arr)O(n log n)In-place sort
Array.BinarySearch(arr, x)O(log n)Requires sorted array
Array.Reverse(arr)O(n)Reverse in place
Array.Fill(arr, val)O(n)Fill all elements
Array.Copy(src, dst, len)O(n)Copy elements
  • Two pointers (palindrome, pair sum, container with most water)
  • Sliding window (max subarray, longest substring)
  • Prefix sums (range sum queries)
  • Binary search (search rotated array, find peak)
  • In-place rearrangement (Dutch flag, move zeroes)

Resizable array backed by internal buffer. Default choice when size is unknown. Amortized O(1) append at end.

Capacity vs Count
Count = elements stored. Capacity = allocated slots. Add() doubles capacity when full.
var result = new List<int>();
for (int i = 0; i < n; i++)
    result.Add(i * 2);

result.Sort();                          // O(n log n)
int idx = result.BinarySearch(target);  // O(log n), must be sorted
result.RemoveAt(result.Count - 1);         // O(1) at end, O(n) in middle

  • When you need resize but not hash lookup

Key-value store with average O(1) operations. The most used structure in DSA for lookups, counting, caching, and graph adjacency.

// Frequency count
var freq = new Dictionary<char, int>();
foreach (char c in s)
    freq[c] = freq.GetValueOrDefault(c, 0) + 1;

// Two Sum — store complement
var seen = new Dictionary<int, int>();
for (int i = 0; i < nums.Length; i++) {
    int need = target - nums[i];
    if (seen.TryGetValue(need, out int j))
        return new[] { j, i };
    seen[nums[i]] = i;
}

// Memoization with tuple key
var memo = new Dictionary<(int, int), int>();

Last-In-First-Out. Essential for DFS (iterative), parentheses validation, and monotonic stack patterns.

// Valid parentheses
var stack = new Stack<char>();
foreach (char c in s) {
    if (c == '(' || c == '[' || c == '{') stack.Push(c);
    else if (stack.Count == 0 || stack.Pop() != Match(c)) return false;
}
return stack.Count == 0;

// Monotonic stack — next greater element
var st = new Stack<int>();
for (int i = 0; i < nums.Length; i++) {
    while (st.Count > 0 && nums[st.Peek()] < nums[i]) {
        int idx = st.Pop();
        result[idx] = nums[i];
    }
    st.Push(i);
}

First-In-First-Out. The go-to structure for BFS, level-order traversal, and shortest path in unweighted graphs.

// BFS — shortest path in unweighted graph
var queue = new Queue<int>();
var visited = new HashSet<int>();
queue.Enqueue(start);
visited.Add(start);
while (queue.Count > 0) {
    int node = queue.Dequeue();
    foreach (int neighbor in graph[node]) {
        if (visited.Add(neighbor))
            queue.Enqueue(neighbor);
    }
}

// Level-order tree traversal
var q = new Queue<TreeNode>();
q.Enqueue(root);
while (q.Count > 0) {
    int levelSize = q.Count;
    for (int i = 0; i < levelSize; i++) {
        var node = q.Dequeue();
        if (node.left != null) q.Enqueue(node.left);
        if (node.right != null) q.Enqueue(node.right);
    }
}
MethodComplexityDescription
Enqueue(x)O(1)Add to rear
Dequeue()O(1)Remove from front
Peek()O(1)View front element
CountO(1)Queue size
  • BFS on graphs and grids
  • Level-order tree traversal
  • Shortest path (unweighted)
  • Sliding window max (with deque)
  • Topological sort (Kahn's with queue)

Min-heap by default (.NET 6+). Always returns smallest priority first. Use for Top K, merge K lists, Dijkstra.

Max-Heap Trick
Negate priorities for max-heap: Enqueue(item, -priority). Or pass Comparer.Create((a,b) => b.CompareTo(a)).
// Top K largest — min-heap of size k
var pq = new PriorityQueue<int, int>();
foreach (int num in nums) {
    pq.Enqueue(num, num);
    if (pq.Count > k) pq.Dequeue();
}

// Dijkstra's algorithm
var dist = new PriorityQueue<int, int>();
dist.Enqueue(source, 0);
while (dist.Count > 0) {
    dist.TryDequeue(out int node, out int d);
    if (d > best[node]) continue;
    foreach (var (next, weight) in edges[node])
        dist.Enqueue(next, d + weight);
}
MethodComplexityDescription
Enqueue(elem, priority)O(log n)Insert with priority
Dequeue()O(log n)Remove min priority
Peek()O(1)View min without removing
TryDequeue(out e, out p)O(log n)Safe dequeue
CountO(1)Heap size
  • Top K frequent/largest/smallest
  • Merge K sorted lists
  • Dijkstra shortest path
  • Find median from data stream
  • Task scheduler, meeting rooms

Doubly linked list in BCL. C# has no built-in Deque — use LinkedList for O(1) add/remove at both ends.

No Built-in Deque
Use LinkedList<T> with AddFirst/AddLast/RemoveFirst/RemoveLast for sliding window max/min problems.
// Deque for sliding window maximum
var deque = new LinkedList<int>(); // stores indices
for (int i = 0; i < nums.Length; i++) {
    while (deque.Count > 0 && nums[deque.Last.Value] < nums[i])
        deque.RemoveLast();
    deque.AddLast(i);
    if (deque.First.Value <= i - k) deque.RemoveFirst();
    if (i >= k - 1) result.Add(nums[deque.First.Value]);
}

First / LastO(1)Access ends
AddAfter/Before(node, x)O(1)Insert relative to node
  • Sliding window max/min (deque)
  • LRU cache (with Dictionary)
  • Palindrome checker with two ends

Custom singly-linked node used in interview problems. Not a BCL type — you define it. Master pointer manipulation: fast/slow, reversal, merge.

public class ListNode {
    public int val;
    public ListNode next;
    public ListNode(int val = 0, ListNode next = null) {
        this.val = val; this.next = next;
    }
}

// Reverse linked list — iterative
ListNode prev = null, curr = head;
while (curr != null) {
    ListNode next = curr.next;
    curr.next = prev;
    prev = curr;
    curr = next;
}
return prev;

// Fast/slow pointer — find middle
var slow = head; var fast = head;
while (fast?.next != null) { slow = slow.next; fast = fast.next.next; }

Binary tree node with left/right children. Traversals: inorder (BST sorted), preorder (serialize), postorder (delete), level-order (BFS).

public class TreeNode {
    public int val;
    public TreeNode left, right;
    public TreeNode(int val = 0, TreeNode l = null, TreeNode r = null) {
        this.val = val; left = l; right = r;
    }
}

// DFS — inorder (left, root, right)
void Inorder(TreeNode node, List<int> res) {
    if (node == null) return;
    Inorder(node.left, res);
    res.Add(node.val);
    Inorder(node.right, res);
}

// Max depth — recursive DFS
int MaxDepth(TreeNode root) =>
    root == null ? 0 : 1 + Math.Max(MaxDepth(root.left), MaxDepth(root.right));

Graphs are built with adjacency lists — not a built-in type. Two common representations: Dictionary for sparse graphs, List[] for dense 0-indexed nodes.

// Adjacency list — Dictionary (sparse)
var graph = new Dictionary<int, List<int>>();
void AddEdge(int u, int v) {
    if (!graph.ContainsKey(u)) graph[u] = new();
    graph[u].Add(v);
}

// Adjacency list — jagged array (0-indexed, dense)
int n = 5;
var adj = new List<int>[n];
for (int i = 0; i < n; i++) adj[i] = new();
adj[0].Add(1); adj[0].Add(2);

// DFS on graph
void Dfs(int node, HashSet<int> visited) {
    visited.Add(node);
    foreach (int nb in graph[node])
        if (!visited.Contains(nb)) Dfs(nb, visited);
}

Dictionary<int, List<int>>O(V+E)Sparse, non-contiguous IDs
List<int>[]O(V+E)0..n-1 nodes, grids

  • Clone graph, word ladder
  • Network delay (Dijkstra)
  • Topological sort, detect cycle

Tree for efficient string prefix operations. O(m) per operation where m = word length. Must implement yourself in C#.

public class TrieNode {
    public Dictionary<char, TrieNode> children = new();
    public bool isEnd;
}

public class Trie {
    private TrieNode root = new();

    public void Insert(string word) {
        var node = root;
        foreach (char c in word) {
            if (!node.children.ContainsKey(c))
                node.children[c] = new TrieNode();
            node = node.children[c];
        }
        node.isEnd = true;
    }

    public bool Search(string word) {

Tracks connected components. Near O(1) amortized with path compression + union by rank. Essential for Kruskal's MST and dynamic connectivity.

public class UnionFind {
    private int[] parent, rank;

    public UnionFind(int n) {
        parent = new int[n];
        rank = new int[n];
        for (int i = 0; i < n; i++) parent[i] = i;
    }

    public int Find(int x) {
        if (parent[x] != x)
            parent[x] = Find(parent[x]); // path compression
        return parent[x];
    }

    public bool Union(int a, int b) {
        int pa = Find(a), pb = Find(b);
        if (pa == pb) return false;
        if (rank[pa] < rank[pb]) (pa, pb) = (pb, pa);
        parent[pb] = pa;
        if (rank[pa] == rank[pb]) rank[pa]++;
        return true;
    }
}
OperationComplexityDescription
Find(x)~O(1)*Get component root
Union(a, b)~O(1)*Merge two sets
Connected(a, b)~O(1)*Find(a) == Find(b)
Count componentsO(n)Track on union
  • Number of connected components
  • Redundant connection, accounts merge
  • Kruskal's minimum spanning tree
  • Graph valid tree check
